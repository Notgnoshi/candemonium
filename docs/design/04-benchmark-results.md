# Takeaways

* The multiplex methods are all pretty close to each other in terms of results
* It appears all backends degrade nicely when the system is under high CPU load
* It doesn't look like I'm going to get absolutely no dropped frames

# TODO

* The multishot io_uring backend could benefit from SINGLE_ISSUER, COOP_TASKRUN, and DEFER_TASKRUN,
  provided the kernel version is sufficient.
* The multishot io_uring backend has a heap allocation in a loop
* The timeout SQE is unnecessary in the multishot backend
* Use large CQ ring size
* The frame loss in the summary should be absolute counts, not percentages
* Analyze results more closely
* Identify more optimization opportunities
